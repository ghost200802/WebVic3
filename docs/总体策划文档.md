# 项目总体策划文档

## 1. 游戏概述

### 1.1 游戏名称
**WebVic3**（暂定）

### 1.2 游戏类型
网页端大型经济模拟策略游戏

### 1.3 核心体验
玩家扮演一个国家/文明的领导者，从石器时代开始，通过发展经济、科技进步、人口管理，带领文明从原始社会走向AI时代。核心体验在于深度经济系统的运筹帷幄。

---

## 2. 游戏愿景

### 2.1 核心设计理念
1. **深度经济模拟**：真实模拟供需关系、生产链条、市场波动
2. **历史演进感**：体验人类文明从采集狩猎到AI时代的技术跃迁
3. **系统涌现**：各系统相互作用产生有趣的策略空间
4. **可理解性**：复杂系统需要清晰的UI呈现

### 2.2 目标受众
- P社游戏爱好者
- 经济/历史模拟游戏玩家
- 喜欢深度策略的玩家

---

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   观察经济状况 ──► 制定政策/投资 ──► 观察结果反馈      │
│        ▲                                    │           │
│        └────────────────────────────────────┘           │
│                                                         │
│   子循环：                                               │
│   • 科技研发：投入资源 ──► 获得新科技 ──► 解锁新生产方式 │

│                                                         │

```

## 5. 核心系统概览

### 5.1 生产系统
- 各时代独特的生产方式
- 生产链：原料 → 中间品 → 最终商品
- 生产效率受科技、劳动力素质影响
- 资源限制与地理分布

### 5.2 市场与价格系统
- 基于供需的动态定价
- 价格影响生产决策
- 国际贸易与价格传导
- 经济周期模拟

### 5.3 人口与就业系统
- 人口增长与结构
- 劳动力分配
- 工资与生活水平
- 社会阶层流动

### 5.4 科技系统
- 时代递进的科技树
- 科技解锁新建筑/生产方式
- 科技扩散与追赶
- 科技研发投入决策

### 5.5 地形地图系统
- 简化的随机地形块，无拓扑关系
- 地块大小统一，但地形组成各异
- 地形类型影响可建设面积与资源分布
- 领土扩张受通讯/控制能力和军事力量限制

### 5.6 运输系统
- 运力为地块内资源，支持地块内的生产活动
- 不同建筑/生产方式对运力有不同需求
- 运力不足时，地块内生产效率下降
- 道路建设提升地块运力上限
- 无跨地块运输需求，简化物流管理

## 6. 领土与地图模型

### 6.1 地块结构

```
玩家领地
├── 地块1（初始）
│   ├── 地形构成：平原60% + 森林30% + 丘陵10%
│   ├── 可建设面积：85单位
│   ├── 矿产资源：铁矿(富集)、煤矿(贫瘠)
│   └── 已建建筑：农田×3、矿场×1、作坊×1
│
├── 地块2（青铜时代解锁）
│   └── ...
│
└── 地块N（随发展逐步解锁）
```

### 6.2 地形类型

| 地形 | 可建设系数 | 产出倾向 | 特殊效果 |
|------|------------|----------|----------|
| 平原 | 100% | 农业、畜牧业 | 无 |
| 森林 | 60% | 木材、狩猎 | 建筑成本+10% |
| 丘陵 | 70% | 采矿、果园 | 防御+20% |
| 水域 | 20% | 渔业、水运 | 需要特殊建筑 |
| 沙漠 | 30% | 石油、太阳能 | 水资源限制 |
| 冰雪 | 25% | 特殊矿产 | 人口效率-20% |
| 沼泽 | 40% | 泥炭、药材 | 疾病风险+10% |

### 6.3 领土扩张机制

| 时代 | 最大地块数 | 解锁条件 | 控制成本 |
|------|------------|----------|----------|
| 石器 | 1 | - | - |
| 青铜 | 3 | 青铜铸造 | +10%/地块 |
| 铁器 | 5 | 道路建设 | +8%/地块 |
| 古典 | 8 | 车轮+驿道 | +6%/地块 |
| 中世纪 | 12 | 封建制度 | +5%/地块 |
| 文艺复兴 | 16 | 地理大发现 | +4%/地块 |
| 工业 | 24 | 铁路网络 | +3%/地块 |
| 电气 | 32 | 电话网络 | +2%/地块 |
| 信息 | 50 | 互联网 | +1%/地块 |
| AI | 100 | AI治理 | +0.5%/地块 |

## 7. 经济模型

### 6.1 商品体系

```
基础资源（石器时代起）
├── 食物：谷物、肉类、蔬菜
├── 原材料：木材、石材、皮革
└── 劳动力

早期商品（青铜-铁器时代）
├── 金属：铜、锡、铁
├── 工具：石器、青铜器、铁器
└── 奢侈品：珠宝、香料

工业商品（工业革命起）
├── 工业原料：煤炭、钢铁、石油
├── 工业品：布匹、机械、化学品
└── 消费品：服装、家具、食品加工品

现代商品（信息时代起）
├── 电子元件：芯片、电路板
├── 高科技产品：计算机、通信设备
└── 服务：软件、数据、AI服务
```

### 6.2 生产方式演进

| 时代 | 生产方式 | 效率倍率 | 特点 |
|------|----------|----------|------|
| 石器 | 手工采集 | 1x | 依赖自然资源 |
| 青铜 | 手工作坊 | 2x | 专业化工匠 |
| 铁器 | 规模作坊 | 3x | 分工协作 |
| 中世纪 | 行会工场 | 4x | 质量控制 |
| 工业革命 | 蒸汽工厂 | 8x | 机械化 |
| 电气时代 | 流水线 | 16x | 标准化 |
| 信息时代 | 自动化 | 32x | 柔性制造 |
| AI时代 | 智能工厂 | 64x | 自主优化 |

## 9. UI/UX设计原则

### 8.1 信息层级
1. **第一层**：关键指标面板（国库、人口、GDP）
2. **第二层**：分系统详情（生产、市场、科技）
3. **第三层**：深度数据（商品详情、人口结构）

### 8.2 视觉风格
- 地图中心化设计
- 数据可视化优先
- 时代主题配色演变

### 8.3 交互原则
- 点击查看详情
- 拖拽批量操作
- 快捷键支持

---

## 10. 数值设计原则

### 9.1 平衡原则
- 任何生产都有投入产出比
- 科技升级需要真实成本
- 防止滚雪球式失衡

### 9.2 增长曲线
- 指数增长有上限
- 边际效益递减
- 技术突破带来阶跃

### 9.3 时间尺度
- 游戏内1年 = 现实时间约5-10分钟（可调）
- 关键事件可暂停

## 11. 开发阶段规划

### Phase 1: 核心原型（4周）

- 石器时代内容

- 完整四大系统
- 青铜-铁器时代内容
- 基础数据可视化

### Phase 3: 内容填充（8周）
- 全部时代内容
- 场景模式
- 教学系统

### Phase 4: 打磨与优化（4周）
- 性能优化
- UI美化
- Bug修复

## 12. 风险与挑战

| 风险 | 应对策略 |
|------|----------|
| 系统过于复杂 | 渐进式信息展示 |
| 数值平衡困难 | 大量测试 + 参数可调 |
| 性能瓶颈 | 异步计算 + 增量更新 |
| 学习曲线陡峭 | 完善教学系统 |
